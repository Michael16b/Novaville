# Lancement de la documentation Novaville 📚

Nous sommes ravis d'annoncer le lancement de la **documentation officielle de Novaville** !

## Qu'est-ce qui est disponible ?

La documentation couvre maintenant tous les aspects de la plateforme :

### 📘 Guide de démarrage

Un guide complet pour installer et configurer l'environnement de développement, que vous travailliez sur le backend Django ou le frontend Flutter.

<!-- truncate -->

### 🔧 Documentation technique

Architecture détaillée du système, incluant :
- Vue d'ensemble de l'architecture en trois tiers
- Documentation du backend Django
- Documentation du frontend Flutter
- Schéma de la base de données PostgreSQL
- Guide de déploiement

### 🌐 Documentation API

Référence complète de l'API REST avec :
- Endpoints d'authentification (JWT)
- CRUD pour les utilisateurs, événements et rapports
- Exemples de code en Python, JavaScript, Dart
- Guide des bonnes pratiques et sécurité

### 📗 Manuel utilisateur

Guide complet pour les utilisateurs finaux :
- Se connecter et créer un compte
- Créer et gérer des signalements
- Participer aux événements
- FAQ et dépannage

## Technologies utilisées

La documentation est construite avec **Docusaurus**, ce qui nous permet de :

- ✅ Versionner facilement la documentation
- ✅ Supporter plusieurs langues (FR/EN)
- ✅ Indexer et rechercher rapidement
- ✅ Déployer automatiquement sur GitHub Pages
- ✅ Personnaliser l'apparence

## Comment y accéder ?

La documentation est accessible à : **https://Michael16b.github.io/Novaville/**

Vous pouvez également l'exécuter localement :

```bash
cd docs
npm install
npm start
```

## Contribuer

La documentation est open source et les contributions sont les bienvenues !

Pour contribuer :

1. Forkez le dépôt
2. Créez une branche pour vos modifications
3. Éditez les fichiers Markdown dans `docs/docs/`
4. Testez localement
5. Soumettez une Pull Request

Consultez le [guide de contribution](../docs/intro#contribution) pour plus de détails.
